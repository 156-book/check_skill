# 确定性问题识别规则

文件名为兼容已有引用而保留。这里定义高置信度低级语言问题的识别范围，但任何命中项都只能报告，不能直接修改原文件。

## 中文错别字

仅报告以下问题：

- “在/再”“的/得/地”等在上下文中的明显误用。
- “已/己”“未/末”等形近字明显混淆。
- 与上下文语义明显不符，且存在唯一合理替换的汉字。

如果替换不唯一或可能改变技术含义，不报告或说明需要用户补充上下文。

## 英文拼写错误

只报告正文中的明确英文拼写错误。不要把语法、用词偏好或大小写风格当作拼写错误。

代码标识符、API 名称、外部库名、包名、品牌名、URL 和路径中的英文不得报告。

## 重复字词

仅报告明显由复制或输入造成的连续重复。不要报告“人人”“渐渐”“看看”等正常表达、修辞性重复、固定表达、产品名和代码内容。

## 多余空格

仅报告打断中文正常阅读的非必要空格。不要处理：

- 列表、代码和表格的缩进或对齐空格。
- 命令、路径、URL、参数、占位符和标识符中的空格。
- 中英文间距风格。
- Markdown 强制换行所需的行末空格。

## 明显标点错误

仅报告结果明确的问题：

1. 多个句号、逗号或叹号意外连续出现。
2. 完整长句明显缺少结束标点，短标题、标签和短语除外。
3. 同一句中括号、引号或书名号明显不匹配，且修复方式唯一。
4. 句号、问号、叹号、逗号、顿号、分号或冒号错误地出现在一行之首。

不要报告全角半角风格、标点前后空格、可能改变语义的断句选择、特殊数学符号或受保护内容中的标点。

## 输出与决策要求

1. 为每个问题生成编号、文件路径、当前行号、原文、原因和最小建议。
2. 同一句中的多个确定性问题可以合并为一个编号，但要逐项说明。
3. 在第一轮按编号逐个展示，每个编号单独调用一次原生提问工具，提供采用建议、保留原文和自由输入改法。
4. 用户完成所有逐问题决定且确认第二轮计划前，不调用编辑工具、不写回文件、不执行格式化命令。
5. 将每个决定写入第二轮计划；只有第二轮统一确认后才修改文件。
